# Validation Rules

## Required Behavior
- Run available lint, typecheck, and build commands for changed app.
- Record unavailable scripts explicitly.

## Test Rules
- Add unit tests for routing utilities.
- Prefer deterministic graph test fixtures.

## Coverage Rules
- Run coverage when script exists and change scope is significant.

## Completion Rule
Work is incomplete unless validation/docs/security/changelog/TODO gates are reported as pass/fail/unavailable.
