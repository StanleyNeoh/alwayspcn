# Product Planning

## PRD Requirement
Every feature must trace to a concrete requirement and acceptance criteria.

## Required Sections
- Problem and user goals
- Scope and non-goals
- Functional requirements
- Technical constraints
- Risks and mitigations
- Acceptance criteria

## Requirement Format
- ID, statement, rationale, acceptance criteria, and dependency notes.

## Scope Control
Defer nice-to-have functionality to backlog unless needed for MVP routing flow.
