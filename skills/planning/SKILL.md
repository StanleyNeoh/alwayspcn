# Planning Rules

## Filename Convention
Use: `plans/YYYY-MM-DD-HHMMSS-<scope>-plan.md`.

## Standard Plan Format
- Goal
- Inputs
- Assumptions
- Implementation steps
- Validation steps
- Risks and rollback

## New-Project Plan Format
Include scaffold, docs, MCP, validation, and handoff evidence.

## Execution Rules
- Re-open `AGENTS.md` before non-trivial tasks.
- Keep tasks small enough for clear verification.
